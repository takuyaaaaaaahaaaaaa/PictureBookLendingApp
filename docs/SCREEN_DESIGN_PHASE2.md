## 2. タブ構成：かえす ／ かりる ／ 図書

```
┌──────────┬──────────┬──────────┐
│ 📕 かえす  │ 📗 かりる  │ 📚 図書   │   既定タブ＝かえす（本を手に持って来る人が最初）
└──────────┴──────────┴──────────┘
```

- ホーム大ボタン方式ではなく**タブ**を採用：用事の切り替えが常時1タップ、現在地が常に見える、
  押し間違いの復帰も1タップ（iOS標準のタブ文法）
- **かえす・かりる**が主動線。**図書**は蔵書を眺める＋本から返す副動線
- 旧「貸出管理」タブは廃止し、俯瞰機能は「設定 > 貸出状況」へ移設（§5）
- ⚠️ iPadのタブバーは画面上部表示（iPadOS 18+）。67歳基準でタップ領域が十分か**要実機確認**。
  不足なら大きめのカスタムボトムバーに差し替える

## 3. かえすタブ（既定・最頻）

```
かえす
└ 借りている子だけの名前一覧        ← 無借用者は表示しない（一覧が短い）
  ├ 検索・五十音・組フィルタチップ     名前のみ。何を借りているかは出さない
  ├ あおき はると
  ├ いとう さくら
  └ タップ
     ↓
   その子の画面
   ├ 園児の本枠：『ぐりとぐら』期限6/20 [返却]
   ├ おうちの人の本枠：『だいくとおにろく』期限6/20 [返却]
   └ 返却完了（✓Undoカード）→ 自動で一覧に戻る
```

- **返却＝名前1＋返却1の2タップ**（タブ切替が要る場合+1）
- 借りている本が1冊なら、名前タップ→即その本が見える。迷う要素がない
- 返却演出はPhase 1の中央カード＋Undoをそのまま流用

## 4. かりるタブ

```
かりる
└ 全園児の名前一覧（0冊の子も当然表示）
  ├ 検索・五十音・組フィルタチップ
  └ タップ
     ↓
   その子の画面（かえすタブと同じ画面・2枠常設）
   ├ 園児の本枠　　：かりていません [借りる] → 図書選択 → ✓
   ├ おうちの人の本枠：かりていません [借りる] → 図書選択 → ✓
   └ 借りている枠には本と期限を表示（満枠なら[借りる]は出ない）
```

- **貸出＝名前1＋借りる1＋図書選択1の3タップ**。組Picker・利用者Picker・種別選択は存在しない
- 2枠が常に見える＝「2冊借りられる」の周知（発見6）
- 図書選択は図書タブの一覧を選択モードで再利用（表紙画像は好評＝発見8を活かす）

## 5. 図書タブ・設定

- **図書タブ**：蔵書一覧（現行の絵本一覧を呼称変更して維持）。
  手に本を持っている人向けに、本からの返却も可能（タイトル検索→[返却]）。
  本からの貸出は「この本をかりる」→名前一覧→枠確定（貸出フォームは廃止、部品を再利用）
- **設定 > 貸出状況（新設）**：旧・貸出管理タブの俯瞰（全体一覧・組別・延滞フィルタ）を移設。
  和田先生の月末チェック用。保護者の主動線から他人の情報を排除する（発見5）。
  `LoanListContainerView` をほぼそのまま移設。PINロックの要否は未決

## 7. 実装計画

### 再利用できるもの

- `LoanModel.getUserActiveLoans(userId:)` / `returnBook` / `undoReturn`：そのまま
- `SuccessFeedback` / `UndoFeedback`（Phase 1）：返却・貸出の演出
- `LoanListContainerView`：「設定 > 貸出状況」へほぼそのまま移設
- `UserListView` の検索・フィルタ機構：名前一覧のベース
- 図書選択：図書一覧の選択モード再利用
- `UserDetailView`：管理用詳細（履歴付き）として設定内に存続

### 新規に作るもの

- 名前一覧（名前のみ・五十音/検索/組チップ。かえす＝借用者のみ／かりる＝全員の2モード）
- 子の画面（2枠常設・返却/借りるボタン・完了後に自動で戻る）
- 図書選択シート（選択モードのラッパー）
- `ContentView` タブ再構成（かえす/かりる/図書）＋設定への俯瞰移設

### 段取り（小さいPR・各段階で実機確認）

0. **ペーパーテスト**：「『ぐりとぐら』を返しに来ました。最初にどこを押しますか?」等5問を
   田中さん相当の方に確認（タブ構成の妥当性を作る前に検証）
1. 子の画面（2枠表示＋返却）
2. 名前一覧（かえす/かりるの2モード）→子の画面への遷移
3. 「借りる」→図書選択→貸出実行（貸出フォーム廃止・「組の記憶」も削除）
4. タブ再構成（かえす/かりる/図書）＋俯瞰を設定内へ移設＋用語「図書」一括適用
5. 主動線のタイポグラフィ底上げ（発見1・DESIGN_PRINCIPLES §5適用）
