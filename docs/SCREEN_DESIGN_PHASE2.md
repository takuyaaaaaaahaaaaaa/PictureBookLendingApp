## 2. モード構成：かえす ／ かりる（画面上部の大型切替）

```
┌─────────────────────────────────┐
│  ┌──────────────┬──────────────┐ │
│  │  📕 かえす     │  📗 かりる    │ │ ← コンテンツ上部の大型切替（既定＝かえす）
│  └──────────────┴──────────────┘ │
│  （下に、選んだモードの一覧が続く）      │
└─────────────────────────────────┘
```

- **TabViewは使わない**。コンテンツ上部に標準部品（大きなボタン2つ）でモード切替を置く
- **上部配置の理由**：切替は「下に何が並ぶか」を決めるスイッチなので、結果（一覧）より
  上にあるのが因果の順序。スタンド置きのキオスクでは親指リーチより視線の起点が優先
  （下部推奨は手持ちスマホの文脈）。視線は上から下へ流れる
- **大型にする理由**：インタビュー発見7「一番触ってほしいところ大きくしておかないと」。
  動詞ファーストIAでは、この切替が最頻のプライマリ操作
- **標準逸脱ではない**：タブバーを持たないアプリはHIG上合法。切替は標準のButton＋
  マテリアルで構成し、システムチャームの模造はしない（Liquid Glass準拠）
- **図書タブは置かない**：かりるモードの最初の画面が図書一覧（蔵書を眺める場所）を兼ねる
- 旧「貸出管理」タブは廃止し、俯瞰機能は「設定 > 貸出状況」へ移設（§5）
- 管理系（図書登録・利用者管理・組管理）は従来どおり設定（⚙）に集約

### 検討した代替案（戻り先として記録）

- **A：標準TabView（iPadは上部タブ）**：Liquid Glassのデフォルトに最も忠実だが、
  タブバーはDynamic Typeでほとんど育たず、67歳基準・発見7に反して小さい。
  ペーパーテストでCが過剰と分かった場合の戻り先
- **B：`.sidebarAdaptable`**：項目2つに過剰・折りたたみ挙動が混乱のもと。不採用
- **D：フルカスタムボトムバー**：システムチャームの模造＝Liquid Glass逸脱コスト最大。使わない

## 3. かえすモード（既定・最頻）

```
かえす（切替の既定）
└ 借りている子だけの名前一覧        ← 無借用者は表示しない（一覧が短い）
  ├ 検索・五十音・組フィルタチップ     一覧は名前のみ。何を借りているかは出さない
  │  ※検索は名前でも図書タイトルでもヒット（本を手に持って来る人向け）
  └ 名前タップ
     ↓
   その子の画面
   ├ 園児の本枠：『ぐりとぐら』期限6/20 [返却]
   ├ おうちの人の本枠：『だいくとおにろく』期限6/20 [返却]
   └ 返却完了（✓Undoカード）→ 自動で一覧に戻る
```

- **返却＝名前1＋返却1の2タップ**。返却演出はPhase 1の中央カード＋Undoを流用
- プライバシー境界＝子の画面の入口。一覧からは誰が何を借りているか分からない（発見5）

## 4. かりるモード（本起点）

```
かりる
└ 図書一覧（表紙グリッド・蔵書を眺める場所を兼ねる）
  ├ 検索・五十音（現行の図書一覧を流用。表紙画像は好評＝発見8）
  ├ 貸出中の本は「かしだしちゅう」表示（借りられないことが分かる）
  └ 本をタップ
     ↓
   「だれがかりますか?」名前一覧（全園児・名前のみ）
   └ 名前タップ
      ↓
    枠の確認画面（2枠常設）
    ├ 園児の本枠　　：かりていません → [この枠でかりる]
    ├ おうちの人の本枠：かりていません → [この枠でかりる]
    │  ※園児枠が満杯でも、おうちの人の枠が空いていればその場で選べる
    └ タップで貸出完了（✓カード）→ 図書一覧に戻る
```

- **貸出＝本1＋名前1＋枠1の3タップ**（＋本を探す検索）。組Picker・利用者Picker・種別選択は存在しない
- 本起点の理由：借りる人は**選んだ本を手に持って**iPadに来る。手にある物から始めるのが自然で、
  「園児の枠は満杯だけど保護者の枠で借りる」という枠の都合は最後にしか分からない
- 枠タップ＝実行確認を兼ねる（誤操作防止）。完了後は✓カード＋Undo
- 将来のカメラ導入（#11/#12/#63）と直結：手の本をスキャン→名前→枠、で検索が消える
