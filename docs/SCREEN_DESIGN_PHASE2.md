## 2. モード構成：かえす ／ かりる（標準TabView・2タブ）

```
┌──────────┬──────────┐
│ 📕 かえす  │ 📗 かりる  │   ← 標準TabView（iPadは上部タブ・既定＝かえす）
└──────────┴──────────┘
```

- **標準TabViewを採用**（A案）。Liquid Glassのデフォルトに乗り、スクロール時の
  表示・非表示挙動・質感・アクセシビリティ・状態管理をシステムから無料で受け取る。
  カスタム切替はこれらを自前管理するランニングコストが発生するため、
  **未検証の推論のために先回りでは払わない**
- 文字サイズはiOS標準の「アプリごとの文字サイズ」最大化＋タイポグラフィ方針
  （DESIGN_PRINCIPLES §5）で底上げ。タブ自体の成長は限定的だが、
  **既定タブ＝かえす**で着地するため、タブ操作が必要なのは借りる人だけ＝頻度は低い
- **図書タブは置かない**：かりるタブの最初の画面が図書一覧（蔵書を眺める場所）を兼ねる
- 旧「貸出管理」タブは廃止。俯瞰はかえすタブの「きげんすぎのみ」フィルタが兼ねる（§3・§5）
- 管理系（図書登録・利用者管理・組管理）は従来どおり設定（⚙）に集約

### フォールバック（ペーパーテスト・実機で証拠が出た場合のみ）

- **C：コンテンツ上部の大型切替**：タブが67歳基準で小さすぎると**実際に確認された**場合、
  TabViewをやめてコンテンツ上部に標準部品の大型切替を置く。根拠になりうる証拠は
  発見7「一番触ってほしいところ大きくしておかないと」と上→下の視線導線。
  ただし現時点では未検証の推論のため採用しない（A⇄Cは中身の2画面が同じで移行が軽い）
- **B：`.sidebarAdaptable`**：項目2つに過剰・折りたたみ挙動が混乱のもと。不採用
- **D：フルカスタムボトムバー**：システムチャームの模造＝Liquid Glass逸脱コスト最大。使わない

## 3. かえすモード（既定・最頻）

```
かえす（切替の既定）
└ 借りている利用者全員の名前一覧      ← 園児も保護者も表示。無借用者は表示しない
  ├ 検索・五十音・組フィルタチップ     一覧は名前＋「きげんすぎ」マークのみ。書名は出さない
  ├ ［きげんすぎのみ］フィルタ         ← 先生の月末チェック＝これ1タップ（俯瞰を兼ねる）
  │  ※検索は名前でも図書タイトルでもヒット（本を手に持って来る人向け）
  └ 名前タップ（園児の名前でも保護者の名前でもよい）
     ↓
   家庭の画面（どの家族名から入っても同じ画面に着地）
   ├ 園児の本枠：『ぐりとぐら』期限6/20 [返却]
   ├ おうちの人の本枠：『だいくとおにろく』期限6/20 [返却]
   └ 返却完了（✓Undoカード）→ 自動で一覧に戻る
```

- **返却＝名前1＋返却1の2タップ**。返却演出はPhase 1の中央カード＋Undoを流用
- **一覧には保護者の名前も出す**：自分が借りた本を自分の名前で探す保護者が迷子にならない。
  園児名・保護者名のどちらから入っても `UserType.guardian(relatedChildId:)` で家庭を解決し、
  同じ家庭の画面に着地する＝**間違うドアがない**
- 家族で2冊借りていても、1回の入場で両方返せる（家庭単位の着地の利点）
- 一覧に出すのは**名前＋延滞マークまで**。書名・期限は家庭の画面でだけ表示（発見5への配慮はこの線）
- 延滞マークの可視化は園の運用実績（入園管理・現行実装）を根拠に許容（[IA_REVIEW.md](IA_REVIEW.md) §6 追記6）

## 4. かりるモード（本起点）

```
かりる
└ 図書一覧（表紙グリッド・蔵書を眺める場所を兼ねる）
  ├ 検索・五十音（現行の図書一覧を流用。表紙画像は好評＝発見8）
  ├ 貸出中の本は「かしだしちゅう」表示（借りられないことが分かる）
  └ 本をタップ
     ↓
   「だれがかりますか?」名前一覧（全利用者・名前のみ。園児でも保護者でもよい）
   └ 名前タップ
      ↓
    家庭の画面＝枠の確認（2枠常設）
    ├ 園児の本枠　　：かりていません → [この枠でかりる]
    ├ おうちの人の本枠：かりていません → [この枠でかりる]
    │  ※園児枠が満杯でも、おうちの人の枠が空いていればその場で選べる
    └ タップで貸出完了（✓カード）→ 図書一覧に戻る
```

- **貸出＝本1＋名前1＋枠1の3タップ**（＋本を探す検索）。組Picker・利用者Picker・種別選択は存在しない
- 本起点の理由：借りる人は**選んだ本を手に持って**iPadに来る。手にある物から始めるのが自然で、
  「園児の枠は満杯だけど保護者の枠で借りる」という枠の都合は最後にしか分からない
- 枠タップ＝実行確認を兼ねる（誤操作防止）。完了後は✓カード＋Undo
- 将来のカメラ導入（#11/#12/#63）と直結：手の本をスキャン→名前→枠、で検索が消える

## 7. 実装計画

### 再利用できるもの

- `LoanModel.getUserActiveLoans(userId:)` / `returnBook` / `undoReturn`：そのまま
- `SuccessFeedback` / `UndoFeedback`（Phase 1）：返却・貸出の演出
- `LoanListContainerView` の延滞判定・組フィルタロジック：かえすタブ一覧に流用
- 図書一覧（現 `BookListView`）：かりるタブの起点として流用（表紙グリッド化は要検討）
- `UserListView` の検索・フィルタ機構：名前一覧のベース
- `UserDetailView`：管理用詳細（履歴付き）として設定内に存続

### 新規に作るもの

- 名前一覧（名前のみ・園児と保護者の両方。かえす＝借用者のみ／かりる＝全員の2モード）
- 家庭の画面（2枠常設・`relatedChildId`で家庭解決。かえす文脈＝[返却]、かりる文脈＝[この枠でかりる]）
- `ContentView` タブ再構成（かえす/かりる・標準TabView）

### 段取り（小さいPR・各段階で実機確認）

0. **ペーパーテスト**：「『ぐりとぐら』を返しに来ました。最初にどこを押しますか?」等5問を
   田中さん相当の方に確認（標準タブのサイズ感も同時に確認→不足ならC案へ）
1. 家庭の画面（2枠表示＋返却）
2. かえすモード（借用者のみの名前一覧→家庭の画面）
3. かりるモード（図書一覧→名前一覧→枠確認→貸出。貸出フォームと「組の記憶」を削除）
4. タブ再構成（かえす/かりる・標準TabView）＋旧貸出管理タブ廃止＋用語「図書」一括適用
5. 主動線のタイポグラフィ底上げ（発見1・DESIGN_PRINCIPLES §5適用）
